Data Analysis

The first step was to analyse the data and visualize it. So the following Images and Diagrams hopefully lead to a better understanding of the given data. We start off with visualizing the representation of each class. class 1 is the diagnosis "no cancer" and class -1 represents the diagnosis "cancer".

Frequency of Classes

Pie-chart of the distribution of the two classes.

after that the data needs to be normalized, so if there would be values which diverge massivly from others this big values would get a higher representation in calculation. So just to be on the safe side, the dataframe is being normalized. To check if that worked, the data is visualized with heatmaps. If there would be any significant differences of the values, there would be a big difference in the coloration.

Heatmap of the whole data

The first image shows the heatmap of the whole data, which means all features.

Heatmap of the relevant features

The thing is, that not all features are relevant for the classification. To provide that irrelevant features have an impact on the model, which could make it more likely to make errors, we select only the relevant ones. A method within the algorithm does this. The following image shows a heatmap of the normalized relevant features

Histograms of the relevant features

To make sure we only chose the relevant features, an upper border is set, that sets the percentage of class-intersection. If the intersection is below 45%, a feature is seen as relevant for further processing. The following image shows an example of the distribution of such features.
